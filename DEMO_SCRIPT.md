# Demo Script

## 5-Minute Demo Flow

1. Start by positioning the project:
   "This is an ADME strategy report generator for early preclinical decision support."
2. Show the default `scientist` mode first to establish technical depth.
3. Switch to `executive` mode to show how the same analysis becomes shorter and decision-led.
4. Switch to `cro_proposal` mode to show how the same result can be reframed as an assay/work-package discussion.
5. Close by explaining fallback safety:
   the system still runs without SMILES, without RDKit, and without live literature access.

## Recommended First Mode

Start with `scientist` mode.

Why:

- it proves the prototype has real technical depth
- it shows the chemistry, literature, evidence-linking, and confidence layers together
- the other modes are easier to understand once the audience has seen the full baseline

## What To Say By Mode

### Scientist Mode

- explain the chemistry intelligence layer
- point out evidence-linked prioritization and confidence-calibrated prioritization
- show that the report is not only listing assays, but explaining why they matter

### Executive Mode

- emphasize top priorities, highest-confidence action, main uncertainty, and next-step plan
- show that the same analysis can be compressed for non-specialist decision-makers

### CRO Proposal Mode

- emphasize first-pass experiments, confirmatory follow-up, and optional exploratory work
- frame the output as a practical assay package discussion rather than a technical deep dive

## Highlights To Emphasize

- same underlying analysis, multiple audience-aware report views
- fallback-safe architecture
- chemistry + literature + evidence + confidence in one artifact
- decision-ready framing instead of just raw technical output

## Boundaries To State Up Front

- the system is heuristic, not a validated predictive engine
- confidence labels are not statistical certainty
- it does not perform full metabolite enumeration
- synthetic demo compounds can have weaker literature anchoring than real compounds
- this is a Stage 1 prototype, not a finished end-user product
